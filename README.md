# 東大山豬盃程式競賽（NTTU_PC）籌備與執行指南

- 版本：1.0
- 適用對象：競賽總召、出題團隊
- 目的：標準化籌備流程，確保競賽傳承不中斷。


## 一、年度時程規劃（Timeline）

舉辦一場程式競賽，建議提前 **4–5 個月** 啟動規劃。

### 1. 第一階段：初期規劃（ 12-1 月）

**確認日期：**

* 查詢系上行事曆，避開期中考與重大活動。
* 務必先請示 **系主任、福哥** 確認系上資源與時間支持。
* 校際協調（若有外校參與），需確認對方行事曆。

**確定賽制與組別：**

* 是否開放高中生組？
* 是否開放外校組？
* 年齡與年級限制（如：大一組、公開組）。

### 2. 第二階段：宣傳與報名（賽前 1.5 個月）

**宣傳啟動：**

* 校內：寄信給 **系辦、校長室** 邀請或尋求贊助。
* 社群：Dcard、Line 群組、FB 社團（東大程式競賽討論區）、系學會粉專。

**報名系統上線：**
建議使用 Google 表單或系上報名系統。

### 3. 第三階段：題目與餐飲（賽前 1 個月～2 週）

* **題目定稿：** 期中考前需完成所有題目設計與測試。
* **餐飲預訂：** 統計報名人數後預訂午餐與點心。


## 二、場地設備與工作人員檢核表（SOP）

### 1. 場地與硬體

| 檢核項目        | 說明                         | 狀態 |
| --------------- | ---------------------------- | ---- |
| 電腦教室        | 理工 C 棟電腦教室（需確認還原卡設定、編譯器環境） | ☐ |
| 伺服器（Server） | 比賽評判系統（DOMjudge 或其他），需壓力測試 | ☐ |
| 列印設備        | 用於列印題目卷、密碼條                | ☐ |
| 氣球與裝飾       | 氣球、打氣筒、瓶子       | ☐ |
| 投影設備        | 播放記分板（Scoreboard）     | ☐ |


### 2. 工作人員規劃（Staff）

假設總參賽隊伍數為 **n**，建議人力配置如下：

* **總召（1 人）：** 統籌所有事務，突發狀況決策。
* **監題／裁判（1–2 人）：** 負責題目釋疑（Clarification）、評判。
* **氣球派送人員（n／6 人）：** 看氣球訊息、吹氣球、綁氣球、派送氣球。
* **場務／顧場（2-3 人/場）：** 協助登入、硬體問題、維持秩序。
* **攝影／機動（1–2 人）：** 活動紀錄、提飯。


## 三、出題與測資設計規範（Problem Setting）

### 1. 出題流程

1. **找想法（Brainstorming）：** 確定核心演算法或邏輯。
2. **寫 Solution（Model Solution）：** 出題者需自己 AC 一遍。
3. **生 Sample 測資：** 題目敘述中的範例輸入／輸出。
4. **撰寫題目敘述（Statement）：** 背景故事、輸入輸出格式、限制。
5. **生 Hidden 測資：** 包含 Corner cases、最大資料量。

> 注意：避免 Python 內建模組（如 itertools）秒殺的題目，必要時調整 Time Limit。

### 2. 難度分級建議（Difficulty Levels）

建議配置多一些 **簽到題** ，難度如下：

| 等級      | 類型  | 說明                          |
| ------- | --- | --------------------------- |
| Level 1 | 簽到題 | 簡單邏輯、基礎數學，避免陷阱。             |
| Level 2 | 基礎題 | 題意理解、矩陣操作、自訂排序。             |
| Level 3 | 中階題 | 資料結構（Stack/Queue/Map）、字串處理。 |
| Level 4 | 進階題 | DP、DFS/BFS、Greedy 等演算法。     |
| Level 5 | 魔王題 | 複數演算法混和、困難圖論／數學題。           |


## 四、餐飲準備指南

**時機：** 報名截止後確認數量，並預留工作人員份量。

**建議菜單：**

* 午餐：餐盒、便當。
* **解題Bar（必備）：** 各種零食。
* 賽後點心：甜甜圈 + 飲料（紅茶、奶茶）。
